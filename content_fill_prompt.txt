==============

STRUCTURE RULES (EXTREMELY IMPORTANT)
====================================
- You MUST return a NESTED JSON object.

  "home.hero.headline"
  "about.title"
  "services.items"
- Dotted keys are STRICTLY FORBIDDEN.

VALID EXAMPLE:

  "home": {
    "hero": {
      "headline": "Example headline"

  }
}

INVALID EXAMPLE (DO NOT DO THIS):

  "home.hero.headline": "Example headline"
}

CONTENT RULES
=============
- This is a CONTENT schema, NOT an editor schema.
- Populate the schema with FINAL WEBSITE CONTENT only.
- DO NOT describe fields.

  type, label, max, options, placeholder, hint, description

Every leaf value MUST be one of:
- string
- number
- boolean
- array of strings
- array of objects containing ONLY content values

If a field represents website text → return the actual text.
If a field represents a list → return realistic items.
If a field represents a structured object → fill ALL inner fields.

NO EMPTY FIELDS
===============
- Do NOT leave fields empty unless the schema explicitly allows null.
